# Next.js 게시판 - 상세 페이지 구현하기

## 기본 동작 흐름
1. 메인 페이지에서 글 제목 클릭
2. 상세 페이지로 이동 (URL에 글번호 포함)
3. 상세 페이지에서 해당 글번호에 맞는 데이터 DB에서 조회
4. 조회된 데이터로 상세 내용 표시

## URL 설계 방식
- 예시: `/detail/1` 접속 시 1번 글의 내용 표시
- 문제: 글이 1000개라면 1000개의 페이지 파일이 필요할까?
- 해결: Next.js의 다이나믹 라우팅 기능 활용

## 다이나믹 라우팅 구현
1. `src/app/detail/[글번호]/page.js` 파일 구조 생성
2. `params.글번호`로 URL 파라미터 접근
3. MongoDB에서 해당 ID로 데이터 조회
4. 조회 결과에 따라 콘텐츠 렌더링

## 코드 구현 핵심
- MongoDB ObjectId 처리
- 데이터 없을 때 예외 처리
- Link 컴포넌트로 페이지 간 이동
